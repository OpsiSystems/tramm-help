Data Manager

The heart of the system — all reference data, transactional data, and financial configuration lives here.

Overview

Data Manager is the central app for managing all of Tramm's data. Every other app — planning, execution, tariffing — depends on the data set up here. Data Manager is divided into three sections, accessible from the left panel.

Master Data

Reference data — vehicles, sites, products, operators, clients, and more. This is the foundation everything else is built on.

Transactional Data

Operational records — Orders, Jobs, Trips, Movements. These are the live records created and processed as freight moves.

Transact Extension

Financial configuration — Contracts, Transport Services, Master Charges, and Tariffs for cost and revenue calculations.

What You See on Screen

1 Left panel — navigation divided into Master Data, Transactional Data, and Transact Extension. Select any entity type to load its records in the main area.
2 Main grid — a list of all records for the selected entity type. Click any row to open the record for viewing or editing.
3 Toolbar — add, edit, delete, and transition records. Import and export buttons (where available) allow bulk data management via spreadsheet.

Before You Start — Settings to Configure First

Several Settings must be configured before Data Manager entities will have all their fields and options available. Set these up before loading master data.

Setting Where in Settings Used by
Dimensions Settings › Definitions › Dimensions Vehicle Classes and Products — defines the capacity units (mass, volume, pallets, etc.) available on these entities.
Routing Classes Settings › Routing Definitions › Routing Classes Vehicle Classes — each vehicle class references a routing class that controls how travel distances and times are calculated.
Attributes Settings › Definitions › Attributes Vehicles, Trailers, Sites, Operators, Products — custom attributes used for resource matching and restrictions.
Operating Days Settings › Definitions › Operating Days Site Classes and Sites — defines operating hour templates that can be applied to sites.
Fatigue Rules Settings › Definitions › Fatigue Rules Routing Classes (and therefore Vehicle Classes) — controls driver break requirements in the solver.
Driving Restrictions Settings › Definitions › Driving Restrictions Vehicles — restrictions on where a vehicle may travel.
Entity Types Settings › Definitions › Entity Types Sites, Vehicles, Operators — classifies entities into user-defined types for filtering and reporting.
Currencies Settings › Definitions › Currencies Transact Extension — required before setting up Contracts, Tariffs, and Charges.
System — Set site address as required Settings › General › System Sites — controls whether address is a required field when creating a site.

Recommended Setup Sequence

Master data entities have dependencies on each other. Setting them up in this order avoids having to revisit records after the fact.

1

Configure in Settings first

  • Dimensions
  • Routing Classes
  • Attributes
  • Operating Days
  • Fatigue Rules
  • Entity Types
  • Currencies (for Transact)
2

Fleet & Vehicle setup

  • Vehicle Classes (uses Dimensions, Routing Classes)
  • Vehicles (requires Vehicle Class)
  • Trailers (requires Vehicle Class)
  • Operator Classes
  • Operators (requires Operator Class)
3

Location setup

  • Site Classes (uses Operating Days)
  • Sites (requires Site Class)
  • Regions
  • Lanes (requires Regions)
  • Locations (standalone geo-entities)
  • Transport Segments (requires Sites)
4

Goods & partners

  • Product Groups
  • Products (requires Product Group)
  • Transport Units
  • Clients (optional link to Sites)
  • Transporters
  • Fleets (requires Sites, Vehicles)
5

Routes (if using master routes)

  • Master Routes (requires Sites)
  • Master Route Sets (requires Master Routes)
6

Transact Extension (financial)

  • Contracts (requires Clients, Transporters, Fleets)
  • Transport Services (requires Vehicle Classes)
  • Master Charges
  • Tariffs

Master Data

Master data is reference data — the stable, reusable records that the system draws on for planning and execution. All master data supports import and export via spreadsheet.

Fleet & Vehicles

Class

Vehicle Classes

A vehicle type specification — dimensions, capacity, compartments, routing class, and costing. Individual vehicles inherit values from their class.

Vehicles

A specific, identifiable transport unit — licence plate, home site, operating hours, and default operator and trailer assignments.

Trailers

A passive transport resource attached to a vehicle. Has its own capacity, compartments, and regulatory permits.

Fleets

A collection of vehicles and trailers sharing a home site. Used for transporter allocation and fleet availability tracking.

Operators

Class

Operator Classes

A driver or service agent type, defining skill attributes (e.g., crane operator, hazardous goods driver).

Operators

A specific individual — operating hours, contact info, regulatory permits, and an optional link to a system user account.

Locations

Class

Site Classes

A location type specification — operating hours, entry/exit durations, and restrictions that sites can inherit.

Sites

Any location a vehicle can visit — depot, delivery point, client site, warehouse. Has operating hours, GPS coordinates, and optional loading docks.

Regions

A geographic area defined by a polygon, postal codes, or coordinates. Used for tariff lane calculations and site grouping.

Lanes

A transportation route between two regions, used primarily for tariff calculations. Can be bidirectional.

Locations

Reusable standalone geographic places with loading docks. Sites can reference a Location entity rather than embedding coordinates directly, allowing multiple sites to share a location.

Transport Segments

A predefined transport leg between two sites — specifies mode, distance, duration, and eligible vehicle classes. Used in crossdock and multi-modal pipelines.

Routes

Master Routes

A route template with ordered stop legs. Controls the sequence of stops on a trip. Used for recurring routes and automated trip generation.

Master Route Sets

A collection of master routes with schedules. When a schedule fires, the system automatically generates trips from the associated routes.

Goods

Product Groups

A goods category type with transport restrictions — specifies what kind of vehicle or compartment the product requires.

Products

A transportable goods item with defined size dimensions, loading rate, and stackability rules. Belongs to one or more product groups.

Transport Units

A container for goods (e.g., pallet, crate, IBC). Has size, capacity, and a product group association.

Partners

Clients

A customer who receives goods or services. Can make orders and hold contracts. Has contact info, location, and order cutoff time schedules.

Transporters

An external transport service provider (carrier or haulier) that delivers on behalf of the organisation. Associated with one or more fleets.

Transactional Data

Transactional data — Orders, Jobs, Trips, and Movements — can be viewed and edited in Data Manager, but these records are primarily created and managed through the operational apps (Order Manager, Planner, Execution Manager). For a full explanation of what these records mean and how they relate to each other, see Key Concepts.

Transact Extension

The Transact Extension covers all financial configuration. These records define how costs and revenues are calculated for every trip. The Transact Extension is only visible to users with the appropriate permissions.

Contracts

Links a client or transporter to a tariff structure. Defines the pricing agreement for a fleet or service type.

Transport Services

Defines a billable service — links vehicle classes and transport units to a service type used in tariff calculations.

Master Charges

Charge definitions — specifies the charge type, schedule, conditions, and values used when calculating costs and revenues.

Tariffs

Pricing structures associated with transporters or clients. Brings together services and charges into a complete tariff for cost and revenue calculations.

Before setting up Transact entities, configure Tariff Engine and Financial Management settings under Settings › Transact.
