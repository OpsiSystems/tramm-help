Overview

Data Manager is the central app for managing all of Tramm's data. Every other app — planning, execution, tariffing — depends on the data set up here. Data Manager is divided into three sections, accessible from the left panel.

Master Data

Reference data — vehicles, sites, products, operators, clients, and more. The foundation everything else is built on.

Transactional Data

Operational records — Orders, Jobs, Trips, Movements. The live records created and updated as operations run.

Transact Extension

Financial configuration — Contracts, Transport Services, Master Charges, and Tariffs for cost and revenue calculations.

What You See on Screen

1 Left panel — navigation divided into Master Data, Transactional Data, and Transact Extension. Select any entity type to load its records in the main area.
2 Main grid — a list of all records for the selected entity type. Click any row to open the record for viewing or editing.
3 Toolbar — standard action buttons for adding, editing, deleting, and transitioning records, plus Import and Export where available. See Standard Toolbar Actions for a full reference.

Before You Start — Settings to Configure First

Several Settings must be in place before Data Manager entities will have all their fields and options available. Set these up before loading master data.

Setting Where in Settings Used by
Dimensions Settings › Definitions › Dimensions Vehicle Classes and Products — defines the capacity units (mass, volume, pallets, etc.) available on these entities.
Entity Types Settings › Definitions › Entity Types Vehicles, Sites, Operators, Regions, and Products — defines what classification values appear in the Type dropdown for each entity. Also controls which custom attributes apply to which entity type.
Attributes Settings › Definitions › Attributes Vehicles, Trailers, Sites, Operators, Products — custom attributes used for resource matching and restrictions between entities.
Operating Days Settings › Definitions › Operating Days Site Classes and Sites — defines operating hour templates that can be applied to sites.
Fatigue Rules Settings › Definitions › Fatigue Rules Routing Classes — controls driver break requirements used by the route solver.
Routing Classes Settings › Routing Definitions › Routing Classes Vehicle Classes — specifies how travel distances and times are calculated per vehicle type. Optional: if not set, the system uses a Default routing class. Configure before go-live for accurate route calculations.
Currencies Settings › Definitions › Currencies Transact Extension only — required before setting up Contracts, Tariffs, and Charges.
System — Set site address as required Settings › General › System Sites — controls whether an address is a required field when creating a site.

Master Data

Master data is reference data — stable, reusable records the system draws on for planning and execution. All master data supports import and export via spreadsheet.

Fleet & Vehicles

Class

Vehicle Classes

A vehicle type specification — dimensions, capacity, compartments, and costing. Individual vehicles inherit values from their class.

Vehicles

A specific, identifiable transport unit — licence plate, home site, operating hours, and default operator and trailer assignments.

Trailers

A passive transport resource attached to a vehicle. Has its own capacity, compartments, and regulatory permits. Only required if using articulated vehicles.

Fleets

A collection of vehicles and trailers sharing a home site. Used for transporter allocation and fleet availability tracking.

Operators — optional

Many Tramm implementations do not use the Operators section at all. Only set this up if you need to track individual drivers and link them to vehicles or trips.
Class

Operator Classes

A driver or service agent type, defining skill attributes — for example, crane operator or hazardous goods driver.

Operators

A specific individual — operating hours, contact info, regulatory permits, and an optional link to a Tramm user account.

Locations

Class

Site Classes

A location type specification — operating hours, entry/exit durations, and restrictions inherited by sites of this class.

Sites

Any location a vehicle can visit — depot, delivery point, client site, warehouse. Has operating hours, restrictions, and an optional link to a Client. A site's geographic coordinates can either be entered directly on the site, or by linking to a shared Location entity (see Advanced & Optional below).

Goods

Product Groups

A goods category with transport restrictions — specifies what kind of vehicle or compartment products in this group require.

Products

A transportable goods item with defined size dimensions, loading rate, and stackability rules. Belongs to one or more product groups.

Partners

Optional

Clients

A customer who receives goods or services. Has contact info, location, and order cutoff time schedules. Not required for planning — a Site is sufficient to plan and execute a delivery. Add Client records when you need to track customer relationships, manage order cutoff schedules, or set up financial contracts in the Transact Extension.

Transporters

An external transport service provider (carrier or haulier) that operates on behalf of the organisation. Associated with one or more fleets.

Routes — optional

Optional

Master Routes

A route template with ordered stop legs. Controls the stop sequence on recurring trips. Only required if using recurring fixed routes.

Optional

Master Route Sets

A collection of master routes with schedules. When a schedule fires, trips are automatically generated from the associated routes.

Advanced & Optional

These entities are only needed for specific scenarios. Most implementations do not require them at initial setup.

Optional

Regions

A geographic area defined by a polygon, postal codes, or coordinates. Used for tariff lane calculations and geographic grouping of sites.

Optional

Lanes

A transportation route between two regions used for tariff calculations. Only needed if using lane-based tariffs in the Transact Extension.

Optional

Locations

A standalone geographic record — coordinates, address, geofence, and loading docks. Used in two scenarios:

1. Shared addresses — when several Sites share the same physical place (e.g., multiple clients in the same industrial park), link them all to one Location instead of duplicating coordinates.

2. Loading Bay Manager — loading docks and bays are defined here, not on the Site. If your operation uses the Loading Bay Manager app for time-slotted dock bookings, you need Location entities.

Optional

Transport Segments

A predefined transport leg between two sites — mode, distance, and duration. Only needed for crossdock and multi-modal operations.

Optional

Transport Units

A container for goods (e.g., pallet, crate, IBC) with size, capacity, and product group association. Only needed if tracking container types on orders.

Transactional Data

Transactional data represents the operational records created and updated as work is planned and executed. These records can be viewed and edited here, but are primarily created through the operational apps.

Orders

A client's request for a delivery, collection, or service at a site. Created in Order Manager.

Jobs

A grouping of orders prepared for planning, taking resource constraints into account. Created in the Planner.

Trips

A vehicle's journey from its start site through all planned stops. Created and managed in the Planner and Execution Manager.

Movements

A higher-level record coordinating orders that move across multiple transport segments and modes (e.g., truck to rail). Each Movement contains segments, and each segment links to its own Trips. Only created when using multi-modal or crossdock planning.

For a full explanation of how Orders, Jobs, and Trips relate to each other, see Key Concepts.

Transact Extension

The Transact Extension covers all financial configuration. These records define how costs and revenues are calculated for every trip. Only visible to users with the appropriate permissions.

Contracts

Links a client or transporter to a tariff structure. Defines the pricing agreement for a fleet or service type.

Transport Services

Defines a billable service — links vehicle classes and transport units to a service type used in tariff calculations.

Master Charges

Charge definitions — specifies the charge type, schedule, conditions, and values used when calculating costs and revenues.

Tariffs

Pricing structures associated with transporters or clients. Brings together services and charges into a complete tariff.

Before setting up Transact entities, configure Tariff Engine and Financial Management in Settings › Transact.
