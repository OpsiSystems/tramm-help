Chora

The route optimisation engine built by Opsi Systems that powers Tramm's planning solver.

Demo

Overview

Chora is Opsi's purpose-built route optimisation solver. It replaces general-purpose solvers and is designed specifically for the complexity of real-world logistics — multiple vehicle types, time windows, capacity constraints, fatigue rules, and loading bay management. Chora runs as the engine behind the Tramm Planner whenever a solver run is triggered.

Supported Job Types

Delivery

Goods are loaded at a depot and delivered to one or more customer sites.

Collection

Goods are picked up from sites and transported to a common offloading site.

Service

A site visit where a service is performed — no goods transfer takes place.

Job Controls

Placement Controls

  • Force exclusive trip — job gets its own route
  • High / low priority
  • Force first stop
  • Force last stop

Grouping Controls

  • Force same trip — multiple jobs on the same route
  • Force different trips — keeps jobs on separate routes (useful for regional planning)
  • Job splitting — oversized jobs automatically split across vehicles

Current Limitations (v1.0)

Single loading site per solve

Each solver run handles one loading site. Use the Solve per Depot option to run independent solves across multiple depots.

No Pickup-and-Delivery (P&D) support

All collections must route to a single common depot. Mixed pickup-and-delivery on the same route is not yet supported.

Manual trailer assignment required

Trailers must be pre-assigned to horses before the solve runs. Automatic trailer assignment is not yet available.
