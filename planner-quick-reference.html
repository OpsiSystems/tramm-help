Planner App — What Every Button Does

A guide for brand new users. Find any button, learn what it does, when to use it, and what to expect after you click it.

The Main Toolbar sits above the Gantt timeline at the top of the Planner. These controls affect how you view the schedule and let you manage trips directly from the timeline. You will use the zoom buttons constantly.

Gantt Toolbar — Actual toolbar buttons (hover over each in Tramm to see its name)

Timeline View Controls

Zoom In

Makes the timeline show a shorter time window so you can see more detail — individual stop times become easier to read.

When to useWhen trips or stops are too close together to read clearly.
What happens nextThe Gantt timeline zooms in. Scroll left and right to navigate.
Zoom Out

Shows a wider time window so you can see more trips at once across the whole day or week.

When to useWhen you want a bird's-eye view of the whole day's schedule.
What happens nextThe Gantt compresses — more trips become visible but with less detail.
Show Selected Stop

Automatically scrolls the Gantt timeline to wherever the stop you have selected is located in time.

When to useWhen you have clicked a stop in the list but cannot see it on the timeline because it is off-screen.
What happens nextThe timeline scrolls to centre on your selected stop.
TipThis is a toggle — click again to turn off auto-scrolling.
Show Loading Date

Scrolls the Gantt to the date when a selected order or job is scheduled to be loaded.

When to useWhen you want to quickly jump to when a job starts rather than searching manually.
What happens nextThe timeline jumps to the loading date of your selected order or job.

Trip Actions (from the Gantt)

Recalculate Trip

Asks the system to recalculate the route timings for the selected trip — for example, after you have added or removed a stop.

When to useAfter changing a trip's stops or time windows and you want the estimated times to update.
What happens nextYou are asked whether to recalculate with new travel times or keep the existing ones. The trip blocks on the Gantt then update.
TipThis button is only active when you have a single trip selected that was not created manually.
Remove Stop

Removes the currently selected stop from the trip directly from the Gantt view.

When to useWhen you can see on the Gantt that a stop should not be part of this trip.
What happens nextThe stop is removed from the trip. The job returns to an unplanned state so you can re-plan it elsewhere.
CautionOnly available for regular stops — not the first load or final offload stop.
Publish Trip

Sends the latest trip information to the driver's mobile device so they have up-to-date instructions.

When to useAfter confirming a trip, or after making last-minute changes that the driver needs to know about.
What happens nextThe trip updates are pushed to the driver's device. Only available once a trip is in Confirmed state (not just Planned).
The Orders tab lists all customer orders for the selected date range. An order is the starting point — it represents a customer's request to move goods from one place to another. Before an order can be scheduled, it must be converted into a Job.

Creating & Editing Orders

Add

Opens a blank order form so you can create a new order from scratch.

When to useWhen a customer request comes in that is not already in the system.
What happens nextA form opens. Fill in the customer, origin, destination, time windows, and any product details, then save.
Edit

Opens the selected order so you can change its details.

When to useWhen an order has incorrect details or the customer has changed their requirements.
What happens nextThe order form opens with existing details pre-filled. Make your changes and save.
Add Duplicate

Creates a new order that is an exact copy of the selected one.

When to useWhen you need to create a similar order for a different day or customer without typing everything again.
What happens nextA copy of the order is created. Edit the details that need to change, then save.
Delete

Permanently removes the selected order from the system.

When to useOnly when an order was created by mistake and has no jobs linked to it yet.
What happens nextA confirmation prompt appears. Confirm to delete permanently. This cannot be undone.
CautionYou cannot delete an order that already has jobs. Remove the jobs first.
Set Time Windows

Opens a time window editor where you can set or adjust pickup and delivery time constraints for one or more selected orders at once.

When to useWhen orders need specific pickup or delivery windows — for example, a customer who can only receive between 08:00 and 12:00.
What happens nextA time window dialog opens. Set your from/to times and save. These windows are carried over when the order is converted to a job.

Planning Actions

Toggle Plannable

Switches an order between Plannable (ready to be scheduled) and Not Plannable (hold it back from scheduling).

When to useWhen an order is confirmed and ready to go — mark it Plannable. If it should wait, mark it Not Plannable.
What happens nextThe order's status updates. Plannable orders appear available for conversion to jobs.
TipYou can select multiple orders and toggle them all at once.
Convert to Jobs

Transforms selected orders into Jobs, which are the units that actually get placed onto trips.

When to useWhen an order is confirmed and Plannable, and you are ready to start scheduling it onto a vehicle.
What happens nextOne or more Jobs are created and appear in the Jobs tab. The order's status updates to show it has been converted.
TipSelect multiple orders and convert them all at once to save time.
Remove Jobs

Removes all jobs that were created from the selected order, returning the order to its unconverted state.

When to useWhen you converted an order to jobs by mistake, or when you need to start the planning for that order over again.
What happens nextThe linked jobs are removed and the order returns to its previous state, ready to be converted again.
CautionIf the jobs are already on a trip, you must unplan them first before removing them.
The Jobs tab is where most of your planning happens. A Job is a specific pickup-and-delivery task, created from an order. Jobs are what you add to trips. A job has an origin, a destination, time windows, and dimensions. Until a job is placed on a trip, it sits here waiting to be planned.

Jobs Panel — Actual toolbar buttons (hover over each in Tramm to see its name)

Creating & Editing Jobs

Add

Creates a new job manually without an order. Use this for ad-hoc movements not linked to a formal customer order.

When to useWhen you have an unplanned, last-minute movement that did not come through the order process.
What happens nextA job form opens. Fill in the origin, destination, and time windows, then save. The job appears in the list ready to be planned.
Edit

Opens the selected job so you can update its details — origin, destination, time windows, or dimensions.

When to useWhen job details are incorrect or have changed since the order was converted.
What happens nextThe job form opens pre-filled. Make changes and save.
Set Time Windows

Sets or adjusts the pickup and delivery time constraints on one or more selected jobs at once.

When to useWhen a customer has specific time constraints that need to be reflected in the planning.
What happens nextA time window dialog opens. Set the from/to times and save. The planner will respect these windows when you build the trip.
Delete

Permanently removes the selected job from the system.

When to useOnly for manually created ad-hoc jobs that are no longer needed.
What happens nextA confirmation appears. Confirm to delete permanently. Jobs linked to orders cannot be deleted this way — use Remove Jobs on the order instead.
Build Trips from Master Route Set

Creates multiple trips at once using a saved set of master routes — useful for running an entire day's planned routes in one action.

When to useWhen your operations follow a fixed daily pattern that has been saved as a Master Route Set — builds all the standard trips for the day in one click.
What happens nextA selector opens to choose the route set. Multiple trips are created automatically, each pre-populated with the standard stops. Review and confirm each one.
TipAsk your system administrator which Master Route Sets are available for your operations.

Job Status Controls

Toggle Plannable

Marks a job as ready to be placed on a trip — or holds it back from planning.

When to useUse Plannable when the job is confirmed and ready. Use Not Plannable if you need to hold it — for example, waiting on customer confirmation.
What happens nextThe job's status updates. Only Plannable jobs can be added to a trip.
Toggle On-Hold

Places a job on hold so it cannot be planned, even if it is marked Plannable. Useful for jobs awaiting approval or with unresolved issues.

When to useWhen a job exists but should not be scheduled yet — waiting on documents, customer approval, or a problem being resolved.
What happens nextThe job is marked On-Hold and excluded from planning. Toggle again to release it.
TipOn-Hold is stronger than Not Plannable — even if Plannable is ticked, an On-Hold job will not be planned.
Unplan

Removes the selected job from its current trip and returns it to an unplanned state so you can plan it differently.

When to useWhen a job has been put on the wrong trip, or you need to rethink the route it is on.
What happens nextThe job is removed from its trip and reappears in the unplanned jobs list. The trip is updated automatically.
Cancel

Cancels the selected job(s) — they will no longer be planned or executed.

When to useWhen a customer cancels a delivery or the movement is no longer required.
What happens nextThe job's status changes to Cancelled. If it was on a trip, it is removed from the trip.
CautionCancelling a job cannot easily be reversed. Check with your team before cancelling.
Transition

Opens the State Transition dialog, which lets you manually move a job through its financial or workflow states — for example, from Pending to In Progress, or to trigger credit processing.

When to useWhen a job needs its financial or integration status updated manually — for example, approving credit or marking financials as processed.
What happens nextThe Job State Transition dialog opens, showing available state machines and the actions you can take. Select an action and click Commit.
TipThis button is used for back-office financial workflows, not day-to-day planning. Ask your team leader before using it for the first time.

Trip Building Actions

Build Trip

The main planning action — takes your selected job or jobs and creates a new trip, where you assign a vehicle, driver, and schedule.

When to useWhen you have one or more jobs ready to be scheduled onto a vehicle. Select the jobs first, then click Build Trip.
What happens nextThe Trip Builder dialog opens. Fill in the trip name, date, start time, vehicle, and driver, then save. The trip appears in the Trips tab.
TipYou can select multiple jobs to combine them into a single trip with multiple stops.
Build Trip from Master Route

Creates a trip using a pre-saved route template, automatically filling in the standard stops and schedule for that route.

When to useWhen you are running a recurring route that your team has already set up as a template — for example, a daily milk run to the same sites.
What happens nextA dialog prompts you to choose a master route template. The trip is pre-populated with the standard stops. Review and confirm.
TipAsk your system administrator which master routes are available for your operations.
The Trips tab shows all trips for the selected date range. A trip is a planned vehicle journey with one or more stops. Here you can edit, confirm, adjust, and manage trips from creation through to completion. Trips move through states: Planned → Confirmed → In Progress → Complete.

Trips Panel — Actual toolbar buttons (hover over each in Tramm to see its name)

Editing Trips

Edit Trip

Opens the trip editor where you can change the trip's name, date, time, stops, and other details.

When to useWhen a trip needs changes — a stop sequence needs reordering, timing is wrong, or details need correcting.
What happens nextThe Trip Builder dialog opens with all current details. Make your changes and save.
Recalculate Trip

Updates the trip's estimated route timings — useful after adding stops, changing the sequence, or when traffic conditions have changed.

When to useAfter editing a trip's stops or sequence and you want accurate estimated arrival times.
What happens nextA prompt asks whether to use new travel times or keep the existing ones. Choose and confirm — the Gantt updates.
Update trip(s) allocation

Opens a dialog to assign or change the vehicle, driver (operator), and transporter for the selected trip.

When to useWhen the originally assigned vehicle or driver is unavailable — for example, a breakdown or sick leave — and you need to reassign.
What happens nextThe allocation dialog opens. Select the new vehicle and/or driver and save. The trip updates immediately.
Add Manual Stop

Adds an extra stop to the trip at a site that is not linked to any job — for example, a fuel stop or a depot check-in.

When to useWhen the driver needs to stop somewhere that is not a customer delivery — such as a weigh bridge, fuel station, or yard check.
What happens nextA dialog opens to select the site and timing. The stop is added to the trip's stop list and appears on the Gantt.
Merge Trips

Combines two or more selected trips into a single trip.

When to useWhen trips that were planned separately can be more efficiently handled by one vehicle in a single run.
What happens nextThe selected trips are merged into one. The combined trip contains all the stops from both. Review the stop sequence after merging.
CautionCheck that the merged trip's vehicle can handle the combined load and that time windows are still met.
Toggle Locks

Locks or unlocks the stops on a trip to prevent accidental changes.

When to useLock a trip once you are happy with it to prevent accidental edits. Unlock it if you need to make changes.
What happens nextAll stops on the trip are locked (or unlocked). Locked stops show a lock indicator and cannot be resequenced or edited until unlocked.
Confirm Plan

Moves the selected trip from Planned to Confirmed status, indicating it is approved and ready to be published to the driver.

When to useOnce you have reviewed the trip, are happy with the vehicle, driver, stops and timing, and want to lock it in.
What happens nextThe trip status changes to Confirmed. You can now use Publish Trip to send it to the driver's device.
TipOnly Confirmed trips can be published. A Planned trip will not appear on the driver's device until it is Confirmed.

Trip Lifecycle Actions

Move Trip

Moves the selected trip to a different date.

When to useWhen a trip needs to be rescheduled — for example, due to a public holiday, vehicle availability, or a customer request to change the delivery date.
What happens nextA date picker opens. Select the new date and confirm. The trip moves to that date on the Gantt.
Update Trip or Stops Actuals

Opens a form to enter real-world execution data — actual start time, actual arrival and departure times at each stop.

When to useWhen a trip has started or completed and you need to capture what actually happened versus what was planned.
What happens nextThe actuals form opens. Enter the real times and save. The trip's status updates to reflect execution progress.
Cancel Trip

Cancels the selected trip. You will be asked to choose a reason and decide what happens to the jobs on it.

When to useWhen a planned trip is no longer going ahead — for example, a vehicle breakdown before the trip starts, or a customer cancellation.
What happens nextA reason selector appears. After confirming, the trip is cancelled. Jobs on the trip return to an unplanned state so you can re-plan them on another trip.
CautionCancellation cannot be reversed. Make sure the trip genuinely cannot go ahead before cancelling.
Abort Trip

Stops a trip that is already in progress — used when something goes wrong mid-journey.

When to useWhen a trip has started but cannot continue — vehicle breakdown en route, driver incident, or road closure.
What happens nextThe trip is marked Aborted. Depending on your system configuration, the remaining jobs may be returned to unplanned, cancelled, or flagged for redelivery. Your dispatcher will guide you on the correct next step.
CautionOnly use Abort for trips that have already started. For trips that have not yet begun, use Cancel Trip instead.

Exporting Data

Export Planned Jobs (CSV)

Downloads a spreadsheet of the trip's jobs and their details in CSV format, which opens in Excel or Google Sheets.

When to useWhen you need to share trip details with someone who does not have access to Tramm, or to attach to a report.
What happens nextA CSV file downloads to your computer. Open it in Excel or any spreadsheet application.
Export Planned Jobs (XLS)

Downloads the same trip data in full Excel format with extended columns and formatting.

When to useWhen you need a more detailed export with additional data fields, or when the recipient uses Excel and needs full formatting.
What happens nextAn XLS file downloads to your computer. Open in Microsoft Excel.
The Trip Builder is the dialog that opens when you click Build Trip or Edit Trip. It is where you define all the details of a trip before saving it to the plan. The dialog has two tabs — Times (where you set the schedule, name, vehicle, and driver) and Sites (where you set the start and end location). Work through both tabs before clicking Commit.

Times Tab — Schedule, Vehicle & Driver

Trip Builder — Times Tab (actual screen)

🏷Name

A name or reference for this trip. The system auto-generates one, but you can change it to something your team will recognise.

When to useChange the auto-generated name if your team uses route names, run numbers, or customer references.
What happens nextThe name appears in the Trips list, on the Gantt timeline, and on any exports.
📅Start Date

The date the vehicle departs on this trip.

When to useSet this when you know the departure date. Without a date, estimated stop times cannot be calculated.
What happens nextThe trip is placed on the Gantt on this date. All estimated stop times calculate from here.
🕐Start Time

The time the vehicle is expected to depart from the first stop.

When to useSet this when you know what time the vehicle departs. Without a start time, estimated arrival times at each stop cannot be calculated.
What happens nextEstimated arrival times at all stops calculate automatically from this time and travel distances.
🚛Vehicle Class

The category of vehicle needed — for example, a 10-ton truck or a refrigerated vehicle. Filters the vehicle list to matching options only.

When to useSelect this first. It narrows down the Vehicle dropdown to only suitable vehicles for the date.
What happens nextThe Vehicle field updates to show only vehicles of that class that are available.
🚛Vehicle

The specific vehicle that will run this trip. This field is optional — you can commit the trip with only a Vehicle Class selected and assign a specific vehicle later.

When to useSelect a specific vehicle when you know exactly which one will be used. Leave blank if you want to finalise the vehicle later.
What happens nextThe vehicle is assigned to the trip. The system will warn you if it is already on another trip that day.
TipOnly Vehicle Class is required. The specific vehicle can be assigned later using Vehicle Allocation in the Trips tab.
👤Operator (Driver)

The driver assigned to this trip. Optional — you can plan without a driver and assign one later.

When to useAssign a driver when you know who will be running the trip. Select an Operator Class first to filter to qualified drivers.
What happens nextThe driver is linked to the trip and can receive it on their mobile device once published.

Sites Tab — Start & End Location

Trip Builder — Sites Tab (actual screen)

📍Start Site

The location where the vehicle begins its journey — usually a depot, warehouse, or yard.

When to useAlways set this. It is typically your own depot or the first pickup point.
What happens nextThe start site appears as the first stop in the Planned Stops list on the left.
TipClick the refresh icon next to the field if the site list does not load.
📍End Site

The location where the vehicle finishes its journey — usually the same depot it departed from, or a different yard.

When to useSet this when the vehicle returns to a specific location after completing all stops. Often the same as the Start Site.
What happens nextThe end site appears as the final stop in the Planned Stops list.

Planned Stops Panel

📋Planned Stops list

Shows the ordered list of stops the vehicle will make on this trip — from start site through all job stops to the end site.

When to useReview this list to check the stop sequence makes sense geographically and fits within time windows.
What happens nextClick any stop to see its details (arrival time, duration, travel time) in the Stop Details panel on the right.
TipDrag stops up or down to resequence them. The system recalculates estimated times automatically.
Unplanned Stops

Jobs that are associated with this trip but have not yet been placed into the stop sequence.

When to useIf jobs appear here, drag them into the Planned Stops list to include them in the route.
What happens nextOnce moved to Planned Stops, the job becomes part of the trip and gets estimated arrival times.

Options (Checkboxes)

☐Disable auto recalculations

Stops the system from recalculating times every time you make a change while building the trip.

When to useTick this when making many changes at once — it speeds things up. Untick when you are done so the system can recalculate everything in one go.
What happens nextEstimated times freeze until you untick the option or click Commit.
☐Is manual trip

Marks this trip as manually planned, meaning the system will not automatically optimise or recalculate its route.

When to useWhen you are building a special trip that must follow a specific sequence and should not be touched by the optimizer.
What happens nextThe trip is excluded from automatic scheduling runs. You retain full manual control.
☐Sync first and last stop times

Automatically aligns the first and last stop times to match the trip's overall start and end times.

When to useTick this when you want the trip's start time and end time to stay in sync with the first and last stops automatically.
What happens nextThe system keeps the times aligned as you make changes to stops or timing.

Bottom Action Buttons

Optimize Trip

Asks the system to automatically find the most efficient stop sequence for this trip based on locations and time windows.

When to useWhen you have added all the stops but are not sure of the best order to visit them.
What happens nextThe system reorders the stops for efficiency and recalculates all estimated times. Review the new sequence before committing.
TipYou can still manually adjust the sequence after optimising if needed.
Apply Master Route

Applies a saved route template to this trip, pre-populating the standard stops and sequence for that route.

When to useWhen this trip follows a regular recurring route that your team has saved as a template.
What happens nextA selector opens to choose the master route. The standard stops are added automatically. You can then add or remove stops as needed.
Commit

Saves all the details you have entered and creates or updates the trip in the system.

When to useOnce you are happy with the name, date, time, vehicle, driver, start/end sites, and stop sequence.
What happens nextThe dialog closes. The trip appears in the Trips tab with a status of Planned and as a block on the Gantt timeline.
TipA Planned trip is not yet sent to the driver. Use Publish Trip from the main toolbar when you are ready to send it.
Close

Closes the Trip Builder without saving any changes.

When to useWhen you opened the builder by mistake or decide not to proceed with this trip.
What happens nextThe dialog closes. No trip is created or changed. Any jobs you had selected remain unplanned.
